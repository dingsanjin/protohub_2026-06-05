# ProtoHub 产品需求文档

## 1. 项目概述

### 1.1 项目背景

ProtoHub 是一个原型托管与在线预览平台，旨在帮助团队和个人便捷地托管、分享和预览各类设计原型及文档。

### 1.2 项目目标

* 提供简单易用的原型文件上传与管理功能

* 支持多种格式的在线预览（HTML/Axure/PDF/draw\.io）

* 支持公开分享与密码授权访问两种模式

* 支持链接有效期管理

* 支持多管理员协作管理

### 1.3 目标用户

* **超级管理员**：平台拥有者，可管理所有管理员账号和全局设置

* **管理员**：被邀请的团队成员，可上传、管理自己的原型文件

* **访问者**：通过分享链接访问原型预览的外部人员，无需注册

### 1.4 部署方式

* 腾讯云轻量应用服务器（上海地域，2核2G）

* 访问形式：`http://公网IP:8080/p/xxx`

* 无需域名备案

***

## 2. 功能需求

### 2.1 认证与授权

#### 2.1.1 管理员登录

* 功能描述：管理员通过账号密码登录系统

* 字段定义：

  * `username`: 用户名（字符串，3-50字符）

  * `password`: 密码（字符串，6-128字符）

* 交互流程：

  1. 访问 `/login` 页面
  2. 输入用户名和密码
  3. 点击登录按钮
  4. 验证成功后跳转到工作台
  5. 验证失败显示错误提示

#### 2.1.2 超级管理员管理

* 功能描述：超级管理员可以创建、删除和禁用其他管理员账号

* 字段定义：

  * `id`: 用户ID（自增整数）

  * `username`: 用户名（唯一，3-50字符）

  * `password`: 密码（加密存储）

  * `role`: 角色（`super_admin` / `admin`）

  * `status`: 状态（`active` / `disabled`）

  * `created_at`: 创建时间

  * `updated_at`: 更新时间

* 交互流程：

  1. 超级管理员登录后，进入管理员管理页面
  2. 查看管理员列表（用户名、角色、状态、创建时间）
  3. 点击"添加管理员"按钮
  4. 输入用户名和初始密码
  5. 点击确认创建
  6. 可对已有管理员进行禁用/启用操作

***

### 2.2 文件管理

#### 2.2.1 文件上传

* 功能描述：管理员可以上传原型文件，支持拖拽和点击选择

* 支持格式：

  * HTML（单文件）

  * ZIP（Axure 导出的压缩包，自动解压）

  * PDF

  * draw\.io（.drawio 文件）

* 限制：

  * 单文件最大 500MB

  * 支持分片上传（大于 10MB 的文件自动分片）

* 交互流程：

  1. 在工作台页面，点击"上传文件"按钮或拖拽文件到上传区域
  2. 选择文件后开始上传，显示上传进度
  3. 系统自动识别文件类型
  4. 如果是 ZIP 包，后台自动解压到独立目录
  5. 上传完成后，文件出现在文件列表中

#### 2.2.2 文件列表

* 功能描述：展示已上传的文件列表

* 展示字段：

  * 缩略图/图标（根据文件类型显示）

  * 文件名

  * 文件类型标签

  * 文件大小

  * 上传时间

  * 访问次数

  * 分享状态（公开/密码保护）

* 视图模式：

  * 卡片视图（默认）：大图标，适合浏览

  * 列表视图：紧凑，适合管理大量文件

* 交互流程：

  1. 进入工作台页面，默认显示卡片视图
  2. 可切换为列表视图
  3. 点击文件卡片可查看详情或进入预览

#### 2.2.3 文件搜索与筛选

* 功能描述：支持多种条件筛选和搜索文件，每个人只能查看自己上传的文件

* 搜索条件：

  * **关键词搜索**：模糊匹配文件名

  * **类型筛选**：全部/HTML/Axure/PDF/draw.io

  * **文件夹筛选**：按文件夹查看（包括"全部"选项）

  * **时间筛选**：今天/本周/本月/自定义时间范围

* 排序方式：

  * 按最近更新时间（默认）

  * 按上传时间

  * 按访问次数

* 交互流程：

  1. 在搜索框输入关键词
  2. 或选择类型/文件夹/时间筛选条件
  3. 或选择排序方式
  4. 列表实时更新匹配结果

#### 2.2.4 文件详情

* 功能描述：查看单个文件的详细信息

* 展示字段：

  * 文件名（可编辑）

  * 文件类型

  * 文件大小

  * 上传时间

  * 更新时间

  * 存储路径

  * 访问统计（访问次数、最近访问时间）

  * 分享设置状态

* 交互流程：

  1. 点击文件卡片进入详情页
  2. 可编辑文件名
  3. 可查看访问统计
  4. 可直接进入预览

#### 2.2.5 文件更新（替换）

* 功能描述：用新文件替换旧文件，保留原分享链接

* 交互流程：

  1. 在文件详情页或操作菜单中选择"更新文件"
  2. 选择新文件上传
  3. 上传完成后，旧文件被替换
  4. 原分享链接保持有效

#### 2.2.6 文件删除

* 功能描述：删除单个或批量文件

* 交互流程：

  1. 在文件列表中选择要删除的文件
  2. 点击删除按钮
  3. 弹出确认对话框
  4. 确认后文件被永久删除，分享链接失效

#### 2.2.7 项目分组

* 功能描述：支持创建文件夹对文件进行分组管理

* 字段定义：

  * `id`: 文件夹ID（自增整数）

  * `name`: 文件夹名称（字符串，1-100字符）

  * `parent_id`: 父文件夹ID（整数，可为空）

  * `created_by`: 创建者ID

  * `created_at`: 创建时间

* 交互流程：

  1. 在工作台页面点击"新建文件夹"
  2. 输入文件夹名称
  3. 创建成功后，文件可拖拽到文件夹中
  4. 点击文件夹可查看其中的文件

***

### 2.3 分享与权限

#### 2.3.1 生成分享链接

* 功能描述：为文件生成短链分享链接

* 链接格式：`http://公网IP:8080/p/{shortId}`

* `shortId`: 6位随机字符串（大小写字母+数字）

* 交互流程：

  1. 在文件操作菜单中选择"分享"
  2. 进入分享设置页面

#### 2.3.2 分享模式设置

* 功能描述：设置分享模式为公开或密码保护

* 模式选项：

  * **公开分享**：任何人知道链接即可访问

  * **密码分享**：需要输入正确密码才能访问

* 交互流程：

  1. 在分享设置页面选择分享模式
  2. 如果选择密码分享，设置访问密码（6-32字符）
  3. 点击保存

#### 2.3.3 链接有效期设置

* 功能描述：设置分享链接的有效期

* 有效期选项：

  * **永久有效**：链接一直有效（除非文件被删除或过期时间被修改）

  * **7天**：7天后链接自动失效

  * **30天**：30天后链接自动失效

  * **自定义日期**：指定具体的过期日期和时间

* 交互流程：

  1. 在分享设置页面选择有效期
  2. 如果选择自定义日期，选择具体时间
  3. 点击保存

#### 2.3.4 复制链接

* 功能描述：一键复制分享链接到剪贴板

* 交互流程：

  1. 在分享设置页面或文件列表中点击"复制链接"按钮
  2. 提示"链接已复制"

***

### 2.4 在线预览

#### 2.4.1 HTML/Axure 预览

* 功能描述：预览 HTML 原型或 Axure 导出的原型

* 实现方式：使用 iframe 嵌入

* 交互流程：

  1. 访问分享链接 `/p/{shortId}`
  2. 系统识别为 HTML/Axure 类型
  3. 使用 iframe 加载原型的 index.html 文件
  4. 支持原型的完整交互功能

#### 2.4.2 PDF 预览

* 功能描述：在线预览 PDF 文件

* 实现方式：使用 PDF.js 库渲染

* 交互功能：

  * 翻页（上一页/下一页）

  * 缩放（放大/缩小/适应宽度/适应高度）

  * 页码跳转

  * 全屏模式

* 交互流程：

  1. 访问分享链接 `/p/{shortId}`
  2. 系统识别为 PDF 类型
  3. 使用 PDF.js 渲染 PDF 文件
  4. 提供翻页和缩放控制

#### 2.4.3 draw\.io 预览

* 功能描述：在线预览 draw\.io 文件

* 实现方式：嵌入 draw\.io 官方 viewer

* 交互流程：

  1. 访问分享链接 `/p/{shortId}`
  2. 系统识别为 draw\.io 类型
  3. 使用 iframe 嵌入 draw\.io viewer，加载文件内容
  4. 支持缩放和拖拽查看

#### 2.4.4 密码验证页面

* 功能描述：对密码保护的链接进行密码验证

* 交互流程：

  1. 访问密码保护的分享链接
  2. 显示密码输入页面
  3. 输入密码并提交
  4. 验证成功进入预览页面
  5. 验证失败显示错误提示，可重新输入

#### 2.4.5 链接过期处理

* 功能描述：对已过期的链接进行处理

* 交互流程：

  1. 访问已过期的分享链接
  2. 显示"链接已过期"页面
  3. 提示联系文件所有者重新生成链接

## 3. 非功能需求

### 3.1 性能要求

* 文件上传：大文件（500MB）上传时间 ≤ 5分钟（取决于网络速度）

* 页面加载：列表页面加载时间 ≤ 2秒

* 预览加载：HTML/Axure 预览加载时间 ≤ 3秒，PDF 预览加载时间 ≤ 5秒

### 3.2 安全要求

* 密码加密：用户密码使用 bcrypt 加密存储

* JWT 认证：管理员登录使用 JWT token，token 有效期 24小时

* 文件安全：上传文件需进行类型校验，防止恶意文件上传

* 密码传输：密码在传输过程中使用 HTTPS（后续配置域名后启用）

### 3.3 可用性要求

* 服务可用性：99.5%（服务器重启、维护时间除外）

* 数据备份：数据库文件每日自动备份

* 文件存储：文件存储在本地磁盘，建议定期备份到 COS

### 3.4 兼容性要求

* 浏览器兼容：支持 Chrome（最新3个版本）、Edge（最新3个版本）、Safari（最新3个版本）

* 移动端兼容：支持 iOS Safari、Android Chrome 移动端访问

## 4. 用户角色与权限

| 角色        | 权限                                   |
| --------- | ------------------------------------ |
| **超级管理员** | 登录系统、管理所有管理员账号、上传/管理/删除所有文件、查看所有文件统计 |
| **管理员**   | 登录系统、上传/管理/删除自己的文件、分享文件、查看自己文件的统计    |
| **访问者**   | 通过分享链接访问预览（无需登录）、输入密码访问密码保护的文件       |

## 5. 业务流程

### 5.1 管理员登录流程

```
访问 /login → 输入账号密码 → 验证 → 成功跳转到工作台 / 失败显示错误
```

### 5.2 文件上传流程

```
选择文件 → 开始上传 → 显示进度 → 完成上传 → 识别文件类型 → 处理文件（ZIP解压）→ 生成记录 → 返回结果
```

### 5.3 分享流程

```
选择文件 → 进入分享设置 → 选择分享模式 → 设置密码（可选）→ 设置有效期（可选）→ 生成短链 → 复制链接
```

### 5.4 预览流程

```
访问 /p/{shortId} → 检查链接有效性 → 检查是否过期 → 检查是否需要密码 → 需要密码则显示密码页面 → 验证密码 → 进入预览页面
```

## 6. 数据字段定义

### 6.1 用户表（users）

| 字段名          | 类型           | 约束                                  | 说明                     |
| ------------ | ------------ | ----------------------------------- | ---------------------- |
| `id`         | INTEGER      | PRIMARY KEY AUTOINCREMENT           | 用户ID                   |
| `username`   | VARCHAR(50)  | NOT NULL UNIQUE                     | 用户名                    |
| `password`   | VARCHAR(255) | NOT NULL                            | 加密后的密码                 |
| `role`       | VARCHAR(20)  | NOT NULL DEFAULT 'admin'            | 角色（super\_admin/admin） |
| `status`     | VARCHAR(20)  | NOT NULL DEFAULT 'active'           | 状态（active/disabled）    |
| `created_at` | DATETIME     | NOT NULL DEFAULT CURRENT\_TIMESTAMP | 创建时间                   |
| `updated_at` | DATETIME     | NOT NULL DEFAULT CURRENT\_TIMESTAMP | 更新时间                   |

### 6.2 文件表（files）

| 字段名               | 类型           | 约束                                  | 说明                            |
| ----------------- | ------------ | ----------------------------------- | ----------------------------- |
| `id`              | INTEGER      | PRIMARY KEY AUTOINCREMENT           | 文件ID                          |
| `user_id`         | INTEGER      | NOT NULL, FOREIGN KEY               | 上传者ID                         |
| `name`            | VARCHAR(255) | NOT NULL                            | 文件名                           |
| `original_name`   | VARCHAR(255) | NOT NULL                            | 原始文件名                         |
| `type`            | VARCHAR(50)  | NOT NULL                            | 文件类型（html/axure/pdf/drawio）   |
| `size`            | INTEGER      | NOT NULL                            | 文件大小（字节）                      |
| `storage_path`    | VARCHAR(500) | NOT NULL                            | 存储路径                          |
| `short_id`        | VARCHAR(50)  | NOT NULL UNIQUE                     | 短链ID                          |
| `share_mode`      | VARCHAR(20)  | NOT NULL DEFAULT 'private'          | 分享模式（public/password/private） |
| `share_password`  | VARCHAR(255) | NULL                                | 分享密码（加密存储，可为空）                |
| `expire_at`       | DATETIME     | NULL                                | 过期时间（可为空表示永久有效）               |
| `visit_count`     | INTEGER      | NOT NULL DEFAULT 0                  | 访问次数                          |
| `last_visited_at` | DATETIME     | NULL                                | 最近访问时间                        |
| `created_at`      | DATETIME     | NOT NULL DEFAULT CURRENT\_TIMESTAMP | 创建时间                          |
| `updated_at`      | DATETIME     | NOT NULL DEFAULT CURRENT\_TIMESTAMP | 更新时间                          |

### 6.3 文件夹表（folders）

| 字段名          | 类型           | 约束                                  | 说明     |
| ------------ | ------------ | ----------------------------------- | ------ |
| `id`         | INTEGER      | PRIMARY KEY AUTOINCREMENT           | 文件夹ID  |
| `user_id`    | INTEGER      | NOT NULL, FOREIGN KEY               | 创建者ID  |
| `name`       | VARCHAR(100) | NOT NULL                            | 文件夹名称  |
| `parent_id`  | INTEGER      | NULL, FOREIGN KEY                   | 父文件夹ID |
| `created_at` | DATETIME     | NOT NULL DEFAULT CURRENT\_TIMESTAMP | 创建时间   |
| `updated_at` | DATETIME     | NOT NULL DEFAULT CURRENT\_TIMESTAMP | 更新时间   |

### 6.4 文件-文件夹关联表（file\_folder）

| 字段名         | 类型      | 约束                       | 说明    |
| ----------- | ------- | ------------------------ | ----- |
| `file_id`   | INTEGER | PRIMARY KEY, FOREIGN KEY | 文件ID  |
| `folder_id` | INTEGER | PRIMARY KEY, FOREIGN KEY | 文件夹ID |

## 7. 页面与路由设计

### 7.1 路由表

| 路由                     | 页面    | 权限    | 说明         |
| ---------------------- | ----- | ----- | ---------- |
| `/login`               | 登录页   | 公开    | 管理员登录      |
| `/`                    | 工作台   | 管理员   | 文件列表与上传    |
| `/files/:id`           | 文件详情  | 管理员   | 文件详情与操作    |
| `/files/:id/share`     | 分享设置  | 管理员   | 设置分享权限和有效期 |
| `/admin/users`         | 管理员管理 | 超级管理员 | 管理管理员账号    |
| `/p/:shortId`          | 预览页   | 公开    | 根据短链ID预览文件 |

### 7.2 页面设计要点

#### 7.2.1 登录页

* 简洁的登录表单（用户名、密码输入框）

* 登录按钮

* 错误提示信息

#### 7.2.2 工作台

* 顶部导航栏（Logo、搜索框、用户信息、退出按钮）

* 侧边栏（文件列表、文件夹树）

* 主内容区（文件卡片/列表、上传按钮）

* 操作栏（视图切换、筛选条件）

#### 7.2.3 预览页

* 顶部工具栏（文件名称、缩放控制、全屏按钮）

* 主预览区（iframe 或 PDF.js 渲染区域）

* 底部状态栏（页码、文件信息）

## 8. 接口设计

### 8.1 认证接口

| 接口     | 方法   | 路径                 | 说明         |
| ------ | ---- | ------------------ | ---------- |
| 登录     | POST | `/api/auth/login`  | 管理员登录      |
| 获取当前用户 | GET  | `/api/auth/me`     | 获取当前登录用户信息 |
| 登出     | POST | `/api/auth/logout` | 登出         |

### 8.2 用户管理接口

| 接口      | 方法     | 路径               | 说明              |
| ------- | ------ | ---------------- | --------------- |
| 获取管理员列表 | GET    | `/api/users`     | 获取所有管理员（仅超级管理员） |
| 创建管理员   | POST   | `/api/users`     | 创建新管理员（仅超级管理员）  |
| 更新管理员   | PUT    | `/api/users/:id` | 更新管理员信息（仅超级管理员） |
| 删除管理员   | DELETE | `/api/users/:id` | 删除管理员（仅超级管理员）   |

### 8.3 文件管理接口

| 接口     | 方法     | 路径                  | 说明           |
| ------ | ------ | ------------------- | ------------ |
| 获取文件列表 | GET    | `/api/files`        | 获取当前用户的文件列表  |
| 上传文件   | POST   | `/api/files/upload` | 上传文件         |
| 获取文件详情 | GET    | `/api/files/:id`    | 获取文件详情       |
| 更新文件   | PUT    | `/api/files/:id`    | 更新文件信息（替换文件） |
| 删除文件   | DELETE | `/api/files/:id`    | 删除文件         |
| 搜索文件   | GET    | `/api/files/search` | 搜索文件         |

### 8.4 文件夹管理接口

| 接口      | 方法     | 路径                 | 说明           |
| ------- | ------ | ------------------ | ------------ |
| 获取文件夹列表 | GET    | `/api/folders`     | 获取当前用户的文件夹列表 |
| 创建文件夹   | POST   | `/api/folders`     | 创建文件夹        |
| 更新文件夹   | PUT    | `/api/folders/:id` | 更新文件夹名称      |
| 删除文件夹   | DELETE | `/api/folders/:id` | 删除文件夹        |

### 8.5 分享接口

| 接口     | 方法   | 路径                         | 说明        |
| ------ | ---- | -------------------------- | --------- |
| 获取分享信息 | GET  | `/api/files/:id/share`     | 获取文件的分享设置 |
| 更新分享设置 | PUT  | `/api/files/:id/share`     | 更新分享设置    |

## 10. 风险与注意事项

### 10.1 技术风险

* **大文件上传**：需实现分片上传，避免内存溢出

* **ZIP 解压**：需检查 ZIP 文件安全性，防止路径遍历攻击

* **PDF 渲染**：大 PDF 文件可能导致内存占用过高

### 10.2 运营风险

* **存储空间**：50GB 系统盘有限，需定期清理或扩展存储（COS）

* **流量限制**：300GB/月流量，大量预览可能超额

### 10.3 安全风险

* **文件类型校验**：严格校验上传文件类型，防止恶意文件

* **密码保护**：分享密码需加密存储，防止泄露

* **访问日志**：记录访问日志，便于追踪异常访问

## 11. 附录

### 11.1 文件类型识别规则

| 文件扩展名             | 识别类型     | 处理方式               |
| ----------------- | -------- | ------------------ |
| `.html`, `.htm`   | HTML     | 直接作为静态文件托管         |
| `.zip`            | Axure    | 自动解压，查找 index.html |
| `.pdf`            | PDF      | 使用 PDF.js 渲染       |
| `.drawio`, `.xml` | draw\.io | 嵌入 draw\.io viewer |

### 11.2 分享链接格式

```
http://{公网IP}:8080/p/{shortId}
```

其中 `shortId` 为 6 位随机字符串，包含大小写字母和数字。

### 11.3 存储目录结构

```
/data/protohub/
  ├── files/              # 文件存储目录
  │   ├── {user_id}/      # 用户专属目录
  │   │   ├── {file_id}/  # 文件目录（ZIP解压后）
  │   │   └── {filename}  # 单文件
  └── database/           # 数据库目录
      └── protohub.db     # SQLite 数据库文件
```
