## 9. API 接口清单（参考）

### 文件管理
- `POST /api/files/upload` - 上传文件
- `GET /api/files` - 分页获取文件列表（支持 folderId/type/search/time/sort）
- `GET /api/files/:id` - 获取文件详情
- `PUT /api/files/:id` - 更新文件名称和/或 folder_id
- `DELETE /api/files/:id` - 删除文件

### 版本管理（新增）
- `POST /api/files/:id/versions` - 上传新版本
- `GET /api/files/:id/versions` - 获取所有版本
- `POST /api/files/:id/versions/:versionId/switch` - 切换到指定版本
- `DELETE /api/files/versions/:versionId` - 删除历史版本

### 文件夹
- `POST /api/folders` - 创建文件夹
- `GET /api/folders` - 获取所有文件夹
- `PUT /api/folders/:id` - 编辑
- `DELETE /api/folders/:id` - 删除

### 分享
- `POST /api/files/:id/share` - 设置分享模式/密码/有效期
- `GET /api/files/:id/share` - 获取分享信息
- `GET /p/:shortId` - 分享预览页

### 预览
- `GET /api/preview/assets/:shortId/*` - 预览静态资源

## 测试结论

### 通过的功能（✅）
1. 后端 TypeScript 编译无错误
2. 前端 TypeScript 编译无错误
3. 数据库迁移正确执行（含 file_versions 表）
4. 文件管理核心功能（上传/重命名/删除/搜索）
5. 文件夹归类功能（前端拖拽 + 后端 API）
6. 文件版本管理（上传新版本/版本列表/切换版本/删除版本）
7. 分享功能（公开/密码/私有三种模式）
8. 多种文件类型预览（HTML/PDF/Draw.io/Axure/ZIP）
9. 访问计数自动递增

### 未发现的问题
- TypeScript 严格模式下的类型定义基本完整
- 拖拽功能交互流畅，folder_id 字段正确从 file_folder 关联表 JOIN 返回
- 版本管理逻辑正确，当前活动版本受保护，历史版本的存储路径互不冲突

### 潜在改进项
- `updateFile` 当前只允许更新 name，可在未来支持更新 share_mode
- 版本管理 Modal 中可增加"备注编辑"功能（API 已存在 PUT /versions/:versionId/note）
- 文件列表中可显示"当前版本号"，帮助用户快速识别文件有多少历史版本
- 可考虑在前端拖拽时增加"拖拽目标文件夹"的动态高亮动画
