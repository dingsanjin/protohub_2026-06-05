# ProtoHub 开发任务拆解文档

## 1. 任务总览

### 1.1 项目目标
完成 ProtoHub 原型托管平台的开发，实现文件上传、管理、分享和在线预览功能。

### 1.2 开发周期预估
| 阶段 | 周期 | 说明 |
|---|---|---|
| 阶段1 | 5个工作日 | 基础框架搭建、认证系统、数据库设计 |
| 阶段2 | 5个工作日 | 文件上传与管理、文件类型识别、存储管理 |
| 阶段3 | 5个工作日 | 在线预览、分享功能、密码验证 |
| 阶段4 | 3个工作日 | 文件夹管理、访问统计、界面优化 |
| 阶段5 | 2个工作日 | 测试、部署、上线 |

### 1.3 优先级定义

| 优先级 | 说明 |
|---|---|
| **P0** | 必须完成，不完成则项目无法正常运行 |
| **P1** | 重要功能，影响用户体验 |
| **P2** | 可选功能，后续可迭代 |

## 2. 阶段1：基础框架搭建（5个工作日）

### 2.2 详细任务说明

#### S1-01：初始化前端项目

**目标**：创建基础的 React + TypeScript + Vite 项目结构

**内容**：
- 使用 `npm create vite@8.5.0 . -- --template react-ts` 初始化项目
- 安装基础依赖：antd, react-router-dom, axios, @types/node
- 配置 tsconfig.json 路径别名

**验收标准**：
- 项目可以正常启动
- 路径别名配置正确
- 基础组件可以正常渲染

#### S1-02：配置前端基础环境

**目标**：搭建前端基础设施

**内容**：
- 配置 React Router v7（createBrowserRouter）
- 创建全局状态管理（AuthContext）
- 封装 Axios 实例（请求拦截器、响应拦截器）
- 创建基础类型定义文件

**验收标准**：
- 路由跳转正常
- Token 自动携带
- 401 自动跳转到登录页

#### S1-03：初始化后端项目

**目标**：创建 Node.js + Express + TypeScript 后端项目

**内容**：
- 创建 server 目录，初始化 npm
- 安装依赖：express, typescript, ts-node, knex, sqlite3, bcrypt, jsonwebtoken, multer, cors
- 配置 tsconfig.json
- 创建服务器启动文件

**验收标准**：
- 后端服务可以正常启动
- TypeScript 编译通过

#### S1-04：数据库设计与初始化

**目标**：创建数据库表结构和初始化数据

**内容**：
- 创建数据库迁移文件（users, files, folders, file_folder 表）
- 创建初始化脚本（创建超级管理员）
- 配置 Knex 连接
- 创建 Repository 基础类

**验收标准**：
- 数据库表创建成功
- 初始管理员账号创建成功
- Repository 可以正常执行 CRUD 操作

#### S1-05：认证中间件开发

**目标**：实现 JWT 认证和角色权限控制

**内容**：
- 创建 JWT 工具函数（生成、验证）
- 创建密码哈希工具函数
- 创建认证中间件（验证 Token）
- 创建角色权限中间件（super_admin/admin 权限控制）
- 创建错误处理中间件
- 创建 CORS 中间件

**验收标准**：
- Token 生成和验证正常
- 密码加密存储
- 无 Token 请求返回 401
- 权限不足返回 403

#### S1-06：登录页面开发

**目标**：实现管理员登录页面

**内容**：
- 创建登录表单组件（用户名、密码）
- 实现登录 API 调用
- 处理登录成功/失败状态
- 登录成功后存储 Token

**验收标准**：
- 登录页面正常显示
- 正确账号密码登录成功
- 错误账号密码显示错误提示
- Token 存储在 localStorage

#### S1-07：管理员管理后端接口

**目标**：实现超级管理员管理其他管理员的功能

**内容**：
- 创建 UsersController
- 创建 UsersService
- 创建 UserRepository
- 实现获取管理员列表接口
- 实现创建管理员接口
- 实现更新管理员接口
- 实现删除管理员接口

**验收标准**：
- 超级管理员可以查看所有管理员
- 超级管理员可以创建新管理员
- 超级管理员可以禁用/启用管理员
- 超级管理员可以删除管理员

#### S1-08：管理员管理前端页面

**目标**：实现管理员管理页面

**内容**：
- 创建管理员列表页面
- 实现添加管理员弹窗
- 实现管理员状态切换
- 实现删除管理员确认

**验收标准**：

## 8. 风险与应对

### 8.1 技术风险

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 大文件上传内存溢出 | 服务崩溃 | 实现分片上传，限制单文件大小 |
| ZIP 解压路径遍历攻击 | 安全漏洞 | 严格检查 ZIP 文件内容，限制解压路径 |
| PDF 渲染性能问题 | 预览卡顿 | 使用 PDF.js 流式渲染，限制 PDF 大小 |

### 8.2 进度风险

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 开发进度延迟 | 上线推迟 | 预留缓冲时间，优先完成 P0 功能 |
| 部署配置错误 | 服务无法启动 | 编写详细部署文档，提前测试部署流程 |

### 8.3 运营风险

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 磁盘空间不足 | 文件无法上传 | 定期清理，配置磁盘告警，后续扩展 COS |
| 流量超额 | 服务中断 | 监控流量使用，配置流量告警 |
